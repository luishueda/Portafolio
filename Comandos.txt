git status (El Inspector):

Qué hace: Te dice qué archivos has modificado, creado o borrado. Es como pasar lista antes de empezar.

Cuándo usarlo: Siempre, antes de cualquier otro comando para ver dónde estás parado.


git add . (El Escenario):

Qué hace: Prepara todos tus cambios para la foto. El punto (.) significa "añade todo lo que cambió".

Cuándo usarlo: Cuando ya terminaste una parte del código y estás listo para guardarla.


git commit -m "tu mensaje" (La Foto):

Qué hace: Toma la foto oficial y la guarda en el historial de tu computadora. El mensaje debe ser breve y descriptivo (ej: "Corregido color de botones").

Cuándo usarlo: Después del add. Es el punto de no retorno local.


git push origin main (La Nube):

Qué hace: Sube tus fotos (commits) desde tu PC a los servidores de GitHub.

Cuándo usarlo: Al final del día o cuando quieras que tus cambios aparezcan en tu perfil de GitHub para que otros los vean.



-------------------------------------------------------------------


git log: Te muestra la lista de todas las "fotos" (commits) que has tomado, quién las tomó y cuándo.

git pull origin main: Trae los cambios de GitHub a tu PC. Úsalo si llegas a editar algo directamente en la web de GitHub (aunque ya quedamos en que es mejor no hacerlo).

git checkout -- nombre_archivo: Si arruinaste un archivo y quieres que vuelva a estar como en la última "foto" que tomaste, este comando descarta tus cambios actuales.
